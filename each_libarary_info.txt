🧠 What each library does
1. 🧩 LangChain

👉 The brain orchestrator of your AI system

Connects LLMs (like Ollama, OpenAI, etc.)
Handles:
prompts
memory
tools (APIs, search, etc.)
agents (decision-making AI)

💡 Example:

from langchain.llms import Ollama
llm = Ollama(model="llama3")

👉 Think: “Glue between everything”
=============================================================

2. 🔌 langchain-community

👉 Extra integrations not in core LangChain

Adds support for:
Ollama
vector DBs
loaders (PDF, websites)
embeddings

💡 You’ll use this for:

Document ingestion
Local LLM connections

👉 Think: “Plugins for LangChain”
==========================================================================
3. 🌐 FastAPI

👉 Backend server for your AI app

Create APIs like:
POST /chat
GET /summary
Very fast + async (production-ready)

💡 Example use:

Your frontend (Streamlit / web app) calls FastAPI

👉 Think: “Engine of your backend”

=============================================================================

4. ⚡ Uvicorn

👉 Runs your FastAPI app

Without it, FastAPI won’t start

💡 Run server:

uvicorn main:app --reload

👉 Think: “Server that runs your backend”

==================================================================================
5. 🎨 Streamlit

👉 Frontend UI (super easy)

Build UI with pure Python
No HTML/CSS needed

💡 Example:

import streamlit as st
st.title("My AI App")

👉 You’ll use this for:

Chat interface
File upload (PDF, video)
Output display

👉 Think: “Frontend for your AI app”

==========================================================================
6. 📦 Pydantic

👉 Defines structured data models

Validates API input/output
Used heavily in FastAPI

💡 Example:

from pydantic import BaseModel

class Query(BaseModel):
    question: str

👉 Think: “Data structure + validation layer”

==============================================================
7. 🔐 python-dotenv

👉 Loads secrets from .env file

Store API keys safely

💡 Example .env:

API_KEY=xyz123
from dotenv import load_dotenv
load_dotenv()

👉 Think: “Secret manager (local)”

1. Black

👉 Automatically formats your code

Fixes indentation
Standardizes spacing
Makes code consistent

💡 Example:

Before:
def add(a,b):return a+b
After running Black:
def add(a, b):
    return a + b

👉 You don’t argue about style — Black decides everything

Run it:

black .



📦 2. isort

👉 Organizes your imports

Groups standard / third-party / local imports
Sorts alphabetically

💡 Example:

Before:
import sys
import numpy
import os
After:
import os
import sys

import numpy

Run it:

isort .




🔍 3. Flake8

👉 Finds errors + bad practices

Unused variables
Syntax issues
Bad naming
Style violations

💡 Example warning:

unused variable 'x'
line too long (120 > 79 characters)

Run it:

flake8 .



🔥 How they work together
Tool	Role
Black	Formats code automatically
isort	Fixes imports
Flake8	Finds issues

👉 Flow:

isort .
black .
flake8 .
⚙️ Best setup (for your projects)

Create config files 👇

pyproject.toml
[tool.black]
line-length = 88

[tool.isort]
profile = "black"
.flake8
[flake8]
max-line-length = 88
exclude = venv
